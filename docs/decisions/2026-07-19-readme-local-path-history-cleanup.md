# README 本机路径历史清理

## 问题现象

英文和中文 README 曾包含开发工作区的 Windows 绝对路径。虽然最新版本可以删除这些内容，但普通提交仍会让旧路径保留在 Git 历史中。

## 根因判断

启动说明直接记录了编写者的工作区路径，没有以仓库根目录和相对路径描述命令。该内容由一个历史提交引入，后续提交没有继续修改 README。

## 关键证据

- 内容搜索确认两份 README 存在固定盘符和工作区目录。
- Git 内容历史搜索确认问题只由一个提交引入。
- 引入提交之后的功能提交不修改 README，因此可以重建引入提交并顺序重放后续提交。

## 修复动作

1. 将运行命令统一约定为从仓库根目录执行。
2. 将管理页依赖安装命令改为基于相对目录的 `npm --prefix` 调用。
3. 将浏览器扩展加载目录改为仓库相对路径。
4. 重建引入问题的历史提交，并在新历史上重放后续功能提交。
5. 使用带明确旧提交租约的 `git push --force-with-lease` 更新远端 `main`，避免覆盖并发更新。

## 后续排查方法

提交 README 或运行文档前，扫描固定盘符、用户目录、用户名和跨项目名称。需要清理历史时，同时检查当前文件和目标远端分支的所有可达提交；推送后再次读取远端分支引用并复查可达历史。

历史重写后，已有克隆需要重新同步；GitHub 缓存、分叉或其他引用中的旧对象不一定立即消失，必要时还需清理对应引用或联系 GitHub Support。
